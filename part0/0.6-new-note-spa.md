# 0.6: New note in single page app

Diagram depicting the situation where the user creates a new note using the
single page version of the app at
<https://studies.cs.helsinki.fi/exampleapp/spa>.

The form has no `action` or `method` attributes. The `spa.js` event handler
calls `e.preventDefault()` so the browser does not reload the page. It creates a
new note object, rerenders the notes list immediately, and sends the note as JSON
to the server with an HTTP POST request to `new_note_spa`. The server responds
with status code 201 created and asks for no redirect, so the browser stays on
the same page and sends no further HTTP requests.

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: User types a note into the text field and clicks Save
    Note right of browser: spa.js event handler calls e.preventDefault()
    Note right of browser: Browser creates note { content, date }, pushes it to the list, and rerenders the notes

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    Note right of browser: Content-Type: application/json
    Note right of browser: Body: { "content": "...", "date": "2019-05-25T15:15:59.905Z" }
    activate server
    Note left of server: Server adds the new note to the notes array
    server-->>browser: 201 Created
    deactivate server

    Note right of browser: No page reload — the browser stays on the same page and sends no further HTTP requests
```
